Features implemented on all 3 HTML pages

  1. Active Navigation Highlight: In all html webpages, the active navigation item is highlighted with a underline and changing color.
  2. Visible Menu Section: The visibility of index.html menu section in index.html is managed based on user interaction with the menu navigation links.
  3. Mandatory Field Requirement: In both register.html and order.html, completion of all fields is mandatory. If any field is left empty or not filled in, an error message will be displayed.
  4. Dynamic Display in Order Form: In the order.html page, when users choose the delivery method, the div containing the delivery address fields option become visible. Moreover, selecting the delivery method as "Pickup" will hide the delivery address fields. Consequently, no error message related to the delivery address will be triggered if the delivery method is not chosen (Same for the payment method).
  5. Automated Billing Address Sync: In order.html, if users check "same as delivery address" after they input their delivery address, it will automatically fill in the billing address. However, if users tick "same as delivery address" before filling in the delivery address fields, an error message will prompt them to first complete the delivery address information.
  6. Card Type Selection: In order.html, users are provided with a selection of credit card types, including Visa, American Express, and Mastercard.
  7. Card Number Validation Based on Card Type: For each credit card type (Visa, American Express, Mastercard) in order.html, there are specific patterns or rules associated with valid card numbers. Examples of the patterns include: Visa and Mastercard have 16 digits, while American Express has 15 digits. If the user enters an invalid card number, an error message will be displayed.
  8. CVV Length Validation: In order.html, when user submit the CVV of credit card, its length must be an exact length of 3 digits.
